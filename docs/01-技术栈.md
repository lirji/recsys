# 推荐系统 · 技术栈选型

> **定位:面向生产的脚手架**。本地能 `docker-compose` 跑通,但架构、接口、可观测、样本回流、AB 实验都按"将来要上生产"的标准预留,不是一次性 demo。
> 约束:Java 微服务为主、无 GPU(向量化走第三方 API,排序模型 CPU 训练)、向量化用第三方免费 API。
> 注:无 GPU 不影响生产可用性——CTR 排序模型(LR/GBDT)本就在 CPU 训练/服务;真要上深度模型再接入推理服务即可。

## 0. 选型总原则

1. **能本地 `docker-compose up` 一键拉起所有依赖**,不依赖云资源。
2. **没有 GPU**:所有"模型"要么是第三方 API(向量化),要么是 CPU 上能跑的轻量模型(LR / GBDT / 协同过滤)。
3. **可替换、可降级**:外部依赖(向量化 API)统一封装成接口,断网/超额时能回退到本地实现。
4. **优先复用你已有的 Java 微服务栈**,方便后续整合进现有体系。

## 2. 关键选型说明(为什么这么选)

### 2.1 为什么用 PostgreSQL + pgvector,而不是专用向量数据库?
- 学习项目数据量小(百万级以内),**一个 Postgres 同时存业务数据 + 向量**,少维护一个组件。
- `pgvector` 支持 HNSW / IVFFlat 索引,余弦/L2 距离,够用且能讲清原理。
- 想升级时再换 Milvus / Qdrant,接口已抽象,改动小。
- > 如果你更习惯 MySQL:业务数据可放 MySQL,向量单独放 pgvector;但**一库到底更简单**,推荐 Postgres-only。

### 2.2 向量化(Embedding):为什么是 Gemini 而不是 Claude?
- **Anthropic / Claude 不提供 embedding 接口**,官方推荐第三方 Voyage AI。
- **Gemini Embedding 有免费额度,且支持多模态**(文本 + 图片),正好满足你"多模态向量化"的诉求。
- 统一封装成 `EmbeddingClient` 接口,实现类:
  - `GeminiEmbeddingClient`(默认,联网)
  - `LocalBgeEmbeddingClient`(降级,本地 ONNX,CPU 可跑)
  - 备选 `VoyageEmbeddingClient` / `OpenAiEmbeddingClient`(都兼容,随时切换)
- **向量维度统一固定**(如 768),写进配置;换模型只改实现类 + 重新灌一次向量。

### 2.3 排序模型:无 GPU 怎么办?
- 排序用的是 **CTR 预估**(预测"用户会不会点"),**LR / GBDT 在 CPU 上训练毫无压力**,根本不需要 GPU。
- 工程上采用业界标准的**"离线训练 / 在线服务"分离**:
  1. **离线**:用 Python + LightGBM 训练(CPU 几分钟),导出为 **ONNX** 格式。
  2. **在线**:Java 用 **ONNX Runtime** 加载 `.onnx` 模型文件打分。
- 这条链路能让你**完整体会真实推荐系统的训练→上线流程**,正是"讲明白原理"的核心。
- 分阶段降低门槛:v1 先用**加权规则打分**(纯 Java,零模型),v2 再上 LR,v3 上 GBDT。

### 2.4 数据集用什么?
- **MovieLens**(电影评分数据集,免费经典):物品=电影,有标题/类型/简介,天然适合做推荐。
  - 入门用 `ml-latest-small`(~10万评分),跑通后换 `ml-25m`。
- 多模态扩展:电影海报图 → Gemini 多模态向量(可选进阶,体现"多模态"亮点)。

### 2.5 哪些组件可以"先不上",降低初期复杂度?
| 组件 | v1 是否必需 | 说明 |
|---|---|---|
| Nacos | 可选 | v1 可用 Spring Boot 直连;模块多了再上注册中心 |
| Kafka | 可选 | v1 行为日志可先写库;做反馈闭环时再上 |
| Flink | 可选 | 实时特征是进阶;v1 用离线批算特征即可 |
| ONNX 排序 | 可选 | v1 先用加权规则,先把召回→展示跑通 |

> **建议节奏**:先用"最小集合"(Postgres+pgvector、Redis、Gemini API)把"召回→展示"跑通,再逐步加 排序 → 反馈 → 实时。

## 3. 本地环境清单

- JDK 21、Maven 3.9
- Docker + Docker Compose
- 一个 **Gemini API Key**(免费申请:https://aistudio.google.com/ )
- (离线训练用)Python 3.10 + lightgbm + onnxmltools
- 中间件全部由 `docker-compose.yml` 提供:PostgreSQL(含 pgvector)、Redis、(可选)Kafka、Nacos

> 所有密钥(Gemini Key 等)走环境变量 / 配置中心,**不硬编码、不进 git**。
