## 0. 现状判断（三路代码级探查结论）

- **算法/模型层已近满配**：12 路召回、9 种排序（v1/onnx/deepfm/dcn/mmoe/ple/din/dien/sim + 双塔粗排 PreRank）、4 种重排（diversity/mmr/dpp/none）、VCG / List-wise / DCO / oCPC / 质量度 / 反作弊 广告全套。代码异常干净，几乎无 TODO 欠债。
- **真正的空白集中在两处**：① **工程护栏**（`docs/06` 自评"部署即崩溃"首因）；② 少数**业界常见但本系统尚缺**的算法 / 广告范式。
- 本清单按四个方向组织：一 工程护栏、二 推荐算法深度、三 广告产品完整度、四 平台/实验/可观测。

## 1. 通用约定（每条工作项都遵循）

- **优雅降级是硬约束**：新增模型/通道/能力必须在缺模型/缺数据/依赖挂时回退到既有行为（参照 `RankRouter` 的 `ObjectProvider` + `isReady()` 兜底范式）。
- **在线/离线特征一致性**：任何新特征/编码器必须在线离线同源（参照 `FeatureAssembler` / `SparseFeatureEncoder` / `SequenceEncoder` 纯函数单一真源）。
- **配置走 `recsys.*` + 环境变量覆盖**，Redis key 一律走 `RedisKeys`。
- **新排序模型的标准落地套路**（已验证 `RankRouter.java`）：
  1. 新建 `XxxRankService implements RankService`（`recsys-rank`）；
  2. `RankRouter` 加 `ObjectProvider<XxxRankService>` + `rank()` 里加 `"xxx"` 分支（照抓 din/ple 分支）；
  3. `RankProperties` 加策略常量；`EvalJob` 支持 `--rank-strategy=xxx`；
  4. Python 训练器 `train/train_xxx.py` → 单文件自包含 `.onnx`（IR9、`dynamo=False`）；
  5. 重训后 **`mvn -pl recsys-rank clean install`**（否则 target/classes 留陈旧资源）。
