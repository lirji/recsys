# 离线评估与严格无泄漏

> **解决什么**:上线前用历史数据证明"推得准"。难点是**评估本身不能穿越**——否则离线数值乐观、上线打脸。本项目 `eval` 作业复用在线流水线评估,并用 `--max-ts` 消除乐观偏置。

## 1. `eval` 作业:复用在线流水线

- 按时间切分留出正样本(`value>=4`),复用**在线 recall→rank 流水线**逐用户评估。
- 指标全家桶 @K:**Precision / Recall / NDCG / MAP / MRR / HitRate / Coverage / Diversity / Novelty**。
- 参数:`--k=10,20,50`、`--recall-only`(逐通道召回评估)、`--rank-strategy=v1|onnx|deepfm|dcn|mmoe|ple|din|dien|sim`、`--max-users`、`--threads`(逐用户并行,默认=CPU 核数)、`--print-split`(只打印切分点 splitTs)。
- 写 `eval/metrics-<ts>.csv`。`run_eval_compare.sh` 在同一 ground-truth 上跑全部策略。

## 2. 指标含义速查

| 指标 | 衡量 |
|---|---|
| Precision@K / Recall@K | 命中率(准 / 全) |
| NDCG@K | 考虑排序位置的增益(位置越靠前权重越大) |
| MAP / MRR | 平均精度 / 首个命中的倒数排名 |
| HitRate@K | 至少命中一个的用户占比 |
| Coverage | 推荐覆盖的物品占全库比(多样性/长尾) |
| Diversity | 结果内部差异度(类目/向量) |
| Novelty | 推荐物品的"冷门度"(反热度) |

**排序模型**另看 **AUC / LogLoss**(离线)。注意 DeepFM 用随机切分,其 AUC 不能与时间切分模型直接比。

## 3. 乐观偏置从哪来

在线 recall→rank 依赖的 CF 倒排、user_embedding、hot 等**是用全历史行为算的**,其中包含了留出期(测试期)的行为 → 评估时"偷看了答案"。这就是 pipeline 乐观偏置。

## 4. 严格 eval(消除偏置)

行为派生作业(`item-cf`/`user-cf`/`swing`/`hot`/`user-embedding`/`bandit-stats`)支持 `--max-ts=<epochSeconds>`:只用切分点前的行为聚合(共享 `BehaviorQuery`,`extract(epoch from ts)<=?`,口径同 `AsOfFeatureBuilder`)。

`run_strict_eval.sh` 一键流程:
```
eval --print-split 取 splitTs
  → 各行为派生作业 --max-ts=splitTs 重建"无泄漏"在线存储
  → eval 评估
  → 用全量重建恢复(不破坏日常存储)
```
**实测 pipeline 偏置仅 ±1~2%**(TWO_TOWER/SEMANTIC 的伪 query 为残余泄漏源)。

## 5. 相关作业

- `data-quality`(E6):embedding 覆盖率 + pCTR 校准偏差 ECE(从 `ad_event` 分桶比实际 CTR)+ 类目分布 PSI(近 N 天 vs 全历史),越阈值 WARN,纯函数 `DataQuality`。
- `publish-reports`:把 `eval/*.csv` 灌 PG `eval_report`(jsonb),供 console-api 读取渲染 ECharts。
- `tune-fusion`:一次 rank pass 出 `(pCTR,pCVR,label)`,网格搜索 `cvrBias/cvrWeight` 的 NDCG。

## 6. 踩坑

1. **不做 strict eval 直接信 pipeline 数值**——CF/vector 通道数值乐观 1~2%(甚至更多如果切分不当)。
2. **DeepFM 随机切分 AUC 与时间切分模型比**——不公平,用 `eval` 任务统一 ground-truth 比。
3. **离线指标好 ≠ 在线 CTR 好**——离线 ground-truth(评分≥4)与线上点击目标有 gap,需 A/B 最终定论。
4. **三处时间切分逻辑重复**(`EvalJob`/`RecCalibrateJob`/`TuneFusionJob`)——改切分口径要同步。

## 7. 面试要点

- **离线评估怎么防穿越**:行为派生存储用 `--max-ts` 只用切分前数据重建。
- **NDCG 为什么比 Precision 更全面**:考虑排序位置的折损。
- **Coverage/Novelty/Diversity**:除了准,还要衡量多样性和长尾,防止只推热门。
- **ECE / PSI**:校准误差(预测 CTR vs 实际)/ 分布漂移,是数据质量巡检的核心指标。
