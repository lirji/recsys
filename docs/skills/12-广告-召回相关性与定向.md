# 广告 · 召回、相关性与定向

> **广告链路的净增量**:广告 pCTR/pCVR **复用推荐的排序模型**,只补"query / 钱 / 校准 / 拍卖 / 出价"。本篇讲广告特有的召回、相关性门槛、智能定向。
> **重要结构**:广告在线主链路已随微服务拆分从 `SearchAdsOrchestrator` 迁到 **`recsys-ad/AdPipeline.java`**(见 [16](16-微服务拆分与gRPC.md))。

## 1. AdPipeline 全链路

`AdPipeline.run()` 顺序(in-process 与 gRPC 共用同一份):
```
广告召回 → 相关性门槛 → 预算/pacing 过滤 → 人群定向
  → score(复用 RankRouter 出 pCTR/pCVR,A6 DFM 可覆盖 pCVR)
  → 批取 titles/ocpcParams → (可选)Listwise sim 预载
  → BiddingService.auction(校准+oCPC+eCPM+GSP/VCG/Listwise)
  → applyDco(创意择优) → applyGuaranteedDelivery(品牌保量)
  → 异步 logImpressions + chargeImpressionBilled(CPM/OCPM 曝光计费)
```
`SearchAdsOrchestrator`(rec-engine)现在只做:① query 理解 ② 广告 A/B(`adVariant`→adBucket+reserve 覆盖)③ 委托 `AdServingGateway.searchAds`。

## 2. 广告召回(`JdbcAdRecallService` / `AdRepository`)

多路召回并合并:

| 路 | 原理 | 数据源 | 降级 |
|---|---|---|---|
| **关键词倒排** | query 分词命中竞价词 | Redis `bidword:inv:{keyword}` ZSet(score=bid) | Redis 空 → DB `kwByDb` |
| **SEMANTIC_AD** | query↔ad 向量余弦 ANN | `ad_embedding`(主库不分片) | 无 embedding → 跳过 |
| **U2A** | 用户向量→广告向量,query 无关个性化 | `user_embedding`→`ad_embedding` | 新用户无向量 → 跳过 |
| 兜底 | 多路兜底 | — | try/catch → 空 |

- `bidword:inv:{keyword}` 是**可服务集**:在线关键词路不回库校验状态,倒排里只有 *广告 active + 审核 approved + 广告主 active* 的竞价词(由写侧 `AdIndexSyncService` 增量维护)。
- `ad_embedding` 从关联 item 的 `item_embedding` 拷贝(`copyEmbeddingFromItem`),**刻意不分片**(向量 ANN 须看全量)。

## 3. 相关性门槛(`RelevanceGate`)

- 召回后、竞价前的**硬过滤**:query↔ad 相关性 < `relevanceThreshold`(默认 0.05)直接丢。守广告体验红线——不相关广告不该因为出价高就展示。
- U2A 以 user↔ad 余弦作相关性依据(同 SEMANTIC_AD)。
- `HOT_AD` 兜底路豁免(恰好通过)。

## 4. 智能定向 Look-alike(A3,`AudienceTargeting`)

- 广告 `audience_id` 非空 → 按人群包 Redis `ad:audience:{id}` 做 `SISMEMBER` 过滤候选;`audience_id` 空 = 不定向、全体可见。
- 人群包由离线 `lookalike` 作业物化:从 `ad_audience_seed` 种子用户的 `user_embedding` 求质心 → pgvector ANN 扩散取 top-N 相似用户 → 写 Redis set `ad:audience:{id}`。
- **Redis 挂 fail-open**(定向失败宁可多展示,不阻断)。

## 5. 相关性 vs 出价 vs 定向的关系

三者是**不同维度的闸门**,依次作用:
1. **定向**:圈定"能看到这条广告的人群"(受众正确性)。
2. **相关性门槛**:圈定"这条广告与当前 query 够相关"(体验红线)。
3. **出价/eCPM**:在通过前两关的候选里排序(商业价值,见 [13](13-广告-竞价与计费机制.md))。
钱不能买穿相关性/定向红线——这是广告系统信任的基础。

## 6. 踩坑

1. **倒排即可服务集**——写侧状态/审核/出价/关键词变化都要增量改倒排,否则在线召回到不可服务广告。
2. **ad_embedding 不能分片**——分片会漏召回(ANN 须全量),且 pgvector `<=>` 不进 ShardingSphere 解析路径。
3. **U2A 对新用户无效**——无 `user_embedding` 则跳过,不报错。
4. **定向 fail-open 的取舍**——可用性优先,但要监控 Redis 命中率防定向失效。

## 7. 面试要点

- **广告召回三路**:关键词倒排(query 精确)、SEMANTIC_AD(query 语义)、U2A(用户个性化,query 无关)。
- **相关性门槛为什么在竞价前**:守体验红线,高出价不能买穿不相关。
- **Look-alike 怎么扩人群**:种子 user_embedding 求质心 → ANN 扩散。
- **可服务集/倒排一致性**:写侧增量维护,在线不回库校验。
