# 广告 · 智能出价与转化建模

> **解决什么**:广告主只想要"转化"(下单/注册),不想手动调 CPC 出价。平台按转化目标**自动出价(oCPC)**,并要解决**转化延迟**带来的反馈偏差。

## 1. oCPC 反馈控制(`OcpcBidder` + `ad-ocpc` 作业)

**在线出价**:
```
bid = targetCpa · pCVR · k
```
- `targetCpa`=广告主目标转化成本,`pCVR`=预估转化率,`k`=校准系数(读 Redis `ad:ocpc:{adv}`,缺则退手动/k=1)。
- 直觉:预期转化价值 = `targetCpa·pCVR`,`k` 修正模型系统性偏差。

**离线反馈控制**(`ad-ocpc`):
```
k_new = clamp( k_old · (目标CPA / 实际CPA)^alpha, min, max )
```
- 实际 CPA 高于目标 → 说明出价过高 → 下调 k;反之上调。这是一个**负反馈控制器**,让实际 CPA 收敛到目标。

## 2. 延迟反馈偏差(核心难点)

**问题**:点击后转化可能几小时~几天才发生。用"当前已观测到的转化"算 CPA,会**系统性高估 CPA**(在途转化还没到)→ k 被过度下调 → 出价越压越低。

### 延迟分布建模(`ad-delay` → `DelayModel`)
把已到达转化的延迟(`转化时刻 − 点击时刻`)拟合成**指数分布** MLE:`λ = 1/mean`,完成曲线 `c(e) = 1 − e^(−λe)`(e=已过去时长)。

### Horvitz-Thompson 纠偏(`ad-ocpc --delay-correct`,默认开)
用 λ 对每个已到达转化按 `1/c(elapsed)` 做 HT 加权,**补回在途转化**得无偏终值再算 CPA:
```
无偏转化数 ≈ Σ  1 / c(elapsed_i)
```
- 直觉:一个点击已过去时间短、完成曲线 c 小 → 它代表的"最终转化"权重大(因为大部分同类转化还没发生)。
- 根治延迟反馈导致的 CPA 高估 / k 过度下调。`--min-completion` 保护(完成度太低不纠偏)。

## 3. 延迟反馈 DFM CVR 模型(A6,Chapelle 2014)

传统 CVR 模型把"点击后还没转化"当负样本,但它可能只是**还没转化**(右删失),导致 pCVR 系统性低估。

`train_dfm.py`:共享 embedding+DNN → **两头**:
- **CVR 头** `p = sigmoid`(去偏 pCVR)。
- **延迟头** `λ = softplus`(转化延迟分布)。

**带删失联合训练**:
```
converted:  Loss = −log p − log λ + λ·d          # d=实际延迟
censored:   Loss = −log( (1−p) + p·e^(−λ·e) )     # e=已过去时长,可能还会转化
```
只导出 CVR 头(去偏 pCVR)。在线 `DfmCvrService`(`recsys-ad`)是 `DeepFmRankService` 镜像,在 `recsys.ad.cvr.enabled` 且就绪时**覆盖**复用 MMoE 头的有删失偏置 pCVR;缺模型退回 MMoE 头(零降级)。

样本来自 `gen-ad-cvr-samples`:从 `ad_event` CLICK↔CONVERSION(join `request_id+ad_id`,`ts<=now()` 右删失)造带删失标签样本,经与在线同源的 `FeatureService`+`FeatureAssembler` 装配特征。

## 4. 多触点归因 MTA(A5,`ad-attribution`)

一次转化前用户可能点/看过多条广告,单一 last-touch 归因不公平。`MultiTouchAttribution`(纯函数)三模型,每次转化恰分出 1.0 信用:
- **linear**:均分。
- **position(U 形)**:首末触点权重高、中间低。
- **time-decay**:`2^(−age/halfLife)`,越近权重越大。

从 `ad_event` 按 `user_id` 重建转化路径,回看窗口内触点(CLICK=点击后 CTA / IMPRESSION=浏览后 VTA,同广告折叠 CLICK 优先)。写 `eval/ad-attribution-<ts>.csv`(last-touch vs MTA 信用 + CTA/VTA 拆分 + credit_delta)。**纯统计报表,不写在线、不进计费**。

## 5. 踩坑

1. **不纠延迟直接算 CPA**——高估 → k 过度下调 → 出价螺旋下降。HT 加权是解药。
2. **CVR 把删失当负样本**——低估 pCVR。DFM 带删失联合训练解决。
3. **MTA 只做报表**——不进计费(计费仍按结算事件),避免归因口径影响钱。
4. **oCPC 缺 k**——退手动出价/k=1,不能因缺系数就不出价。

## 6. 面试要点

- **oCPC 原理**:`bid=targetCpa·pCVR·k`,k 用实际 CPA vs 目标 CPA 负反馈调节。
- **延迟反馈两个层面**:①在线出价的 CPA 统计要 HT 纠偏(`ad-delay`+`ad-ocpc`);②CVR 模型要带删失联合训练(DFM)。
- **Horvitz-Thompson**:按完成曲线倒数加权补回在途转化,得无偏估计。
- **MTA 三模型**:linear/position/time-decay,信用和为 1;只做报表不进计费。
