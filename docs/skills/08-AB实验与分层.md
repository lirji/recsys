# A/B 实验与分层

> **解决什么**:任何推荐/广告改动都要用数据证明有效。本项目实现**分层实验**(多层正交、可组合)+ 确定性分桶 + 统计显著性检验。

## 1. 分层实验(layered experiment)

四层各是独立实验,各有自己的 salt 和 variants,**互相正交、可任意组合**:

| 层 | 切什么 | 默认变体 |
|---|---|---|
| `recall` | 启用的召回路集合(`channels`) | base / plus(plus 含 TWO_TOWER/GENERATIVE/TIGER) |
| `rank` | 排序策略(v1/onnx…)+ 融合权重 | v1 / onnx |
| `rerank` | 重排策略 + 参数 | diversity / mmr |
| `ad` | 广告变体(reserve-price 等覆盖) | base / high-reserve |

recall×rank×rerank = 8 种组合。配置 `recsys.experiment.layers.*`,默认 `enabled=true` 各 50/50。

## 2. 确定性分桶

```
bucket = floorMod( (userId + "|" + salt).hashCode(), 100 )
→ 按 variant 权重累积命中
```
- **确定性**:同 userId + 同 salt 永远同桶。故结果缓存按 userId 即可,无需按 bucket 拆。
- `enabled=false` 时每层取第一个 variant 作基线。
- **`ad` 层单独分桶**:`ExperimentService.adVariant` 独立分桶,**不混入推荐 bucketTag**,`ad_event.ad_bucket` 落库供广告分层 A/B。

## 3. 动态覆盖(不重启改实验)

- Redis `recsys:exp`(`ExperimentOverrideService`),30s 缓存 + `invalidate()`。
- 管理面 `/api/experiment`:GET 快照 / POST 开关·放量·停止 / DELETE 清空。改实验不重启。

## 4. 埋点与归因

- `ExposureLogger` 每次推荐异步写 `user_behavior` 一行 `action=IMPRESSION`,`bucket` 记紧凑分层标签(如 `recall:plus;rank:onnx;rerank:mmr`),冷启动加 `cold;` 前缀,`position` 记 1-based 位次(供 PAL)。
- Redis `expo:{user}:{item}=bucket`(TTL 2h)供点击时回填 bucket 归因。
- 点击来自 `POST /api/behavior` 真实埋点 → `GROUP BY bucket` 算分桶 CTR。

## 5. 统计显著性(`ab-report` + 纯函数 `AbStats`)

`ab-report` 作业基于曝光/点击日志逐桶算 CTR,并对基线桶做:
- **Wilson 95% 置信区间**(小样本比例区间比正态近似稳)。
- **两比例 z 检验** → p 值 / 是否显著(`--alpha` 默认 0.05)/ lift。
- **每臂最小样本量**(检测该 lift 所需,`--power`)。
- **AA 校验**:把同策略两桶当 A/B 跑,若显著 → 分桶/埋点有偏(健康度自检)。

写 `eval/ab-report-<ts>.csv`;`--baseline`/`--alpha`/`--power` 可调。

## 6. 踩坑

1. **按 bucket 拆缓存是多余的**——确定性分桶下同 userId 恒定同桶,缓存 key 用 userId 即可。
2. **IMPRESSION 埋点必须带 bucket + position**,否则无法做分桶 CTR / PAL 闭环。
3. **ad 层不能混入推荐 bucketTag**——否则推荐分桶 CTR 被广告变体污染。
4. **不做 AA 校验直接信 A/B 结果**——分桶哈希偏斜/埋点丢失会造成假显著。

## 7. 面试要点

- **分层实验为什么正交**:各层独立 salt,组合空间大、复用流量。
- **确定性分桶**:hash(userId|salt),同用户恒定,可缓存。
- **Wilson 区间 vs 正态近似**:小样本/极端比例时 Wilson 更稳。
- **AA 测试**:同策略对照,验证分桶/埋点无偏,是上线 A/B 前的必要体检。
- **最小样本量/功效**:能检测多大 lift 取决于样本量,报告要给出。
