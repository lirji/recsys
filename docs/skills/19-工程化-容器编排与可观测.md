# 工程化 · 容器编排与可观测

> **解决什么**:让这套多服务系统"一键本地跑通、可容器化、可上 k8s、可观测、可 CI 守门"。这些是脚手架"面向生产"承诺的兑现。

## 1. 本地一键起(`scripts/dev-local.sh`)

固化了所有 env/端口/启动顺序坑:
- `up`:staggered 起 8 个后端 app(rec-engine→behavior→content→user→advertiser→ad-serving→console→gateway 最后),健康等待(日志 grep + `/actuator/health`)。
- `frontend`:起 Vite → `$GATEWAY_PORT`。
- `status`/`down`/`restart`/`logs <svc>` 同名子命令。
- **内建规避的 4 个坑**:①必须 bash 非 zsh;②网关也要 `RECSYS_SECURITY_ENABLED=false`(否则 `/api/advertiser` 等 401);③rec-engine/advertiser/ad-serving 都要 `PG_JDBC_URL`+`PG_DS1_JDBC_URL`(ad 读侧 ShardingSphere);④staggered 启动避免 pg 连接池风暴。
- 端口冲突时覆盖写 `scripts/dev-local.env`(本机常用:PG 5455、Redis 6380、端口→90xx、Prometheus 19090)。

## 3. 容器化 & k8s

- **一个参数化 Dockerfile** 服务所有 Spring Boot app:`--build-arg MODULE/PORT`,pom-layer 缓存(21 个 pom),`-pl MODULE -am`,运行时 `eclipse-temurin:21-jre` 非 root(1001)。recsys-streaming 除外(需 `--add-opens`)。
- **k8s**(`deploy/k8s/`,kustomize base + dev/prod overlay):三探针,**readiness = `/actuator/health/readiness`**(含 `ModelReadinessHealthIndicator`——模型策略但 ONNX 没加载 → readiness DOWN,k8s 不导流)。rec-engine startupProbe(failureThreshold 30);env `AD_SERVING_MODE=in-process`(kill-switch,直到 money-chain-gate 绿)、`CONTENT/USER_SERVING_MODE=grpc`。

## 4. 可观测(`monitoring/`)

- **Prometheus**:8 个 host target;指标 `recsys_recommend_duration_seconds_*`(P50/P95/P99)、`recsys_ad_duration_seconds_*`、缓存命中率、模型回退率、分桶曝光/点击。
- **告警规则**(`alert-rules.yml`):推荐 SLO 99.5%/P99<200ms 多窗口 burn-rate(14.4×/3×);RecEngineDown、RecommendErrorBudgetBurn Fast/Slow、RankModelFallbackHigh、CacheHitRatioLow、EmptyRecallSpike;广告 SLI(P99<300ms、fill、CTR、eCPM、invalid-click)+ AdServingLatencyP99High、AdFillRatioLow、AdInvalidClickSpike。
- **Alertmanager** → `webhook-logger:5001`(Slack/钉钉/飞书 demo)。
- **Grafana**(:3001):datasource + 2 看板(`recsys-online.json`、`recsys-ads.json`)。
- **Tempo**:OTLP 分布式追踪后端(P2)。
- **console-api BFF 观测**:`SystemHealthService`(并行探 8 app + 2 passive job)、`SystemMetricsService`(PromQL,never throws)、`SystemOverviewService`(静态模块图),供前端总览/诊断/告警页。

## 5. 熔断(Resilience4j)

重外呼加熔断快速失败到降级:
- `GeminiEmbeddingClient`(gemini-embedding)。
- `VectorRecaller` 的 pgvector ANN(pgvector-recall,pg 宕 → VECTOR 路快速返回空、其它路兜底)。
- gRPC 客户端每 RPC 熔断(见 [16](16-微服务拆分与gRPC.md))。
- 配置在 rec-engine `application.yml` 的 `resilience4j.circuitbreaker`。

## 7. 配置热更(三层)

| 机制 | 粒度 | 存储 |
|---|---|---|
| Nacos 配置中心(P4) | `recsys.ad.*`/`recsys.fusion.*` 在线热更不重启 | Nacos dataId(`optional:` 不可达退本地) |

Nacos 消费方每请求读活的 getter,推送触发 `ConfigurationPropertiesRebinder` 就地重绑,**无需 `@RefreshScope`**。

## 8. 踩坑

1. **zsh 起脚本失败**——必须 bash。
2. **网关不关 security 本地 401**——`RECSYS_SECURITY_ENABLED=false`。
3. **本机 9090 被 console 占**——Prometheus 用 19090(`docker-compose.local.yml`)。
4. **重训模型后不 clean install**——ONNX 陈旧,`target/classes` 留旧资源。
5. **readiness 探针含模型就绪**——ONNX 没加载会 DOWN,别用 liveness 代替。

## 9. 面试要点

- **readiness vs liveness**:readiness 含 `ModelReadinessHealthIndicator`,模型没就绪不导流;liveness 只看进程活着。
- **money-chain-gate**:计费相关测试单独 gate,必过才允许拆 ad-serving(kill-switch `AD_SERVING_MODE=in-process`)。
- **多窗口 burn-rate 告警**:快窗(14.4×)+ 慢窗(3×)组合,兼顾灵敏与抗抖。
- **三层配置热更**:Nacos(结构化)/ Redis tuning(轻量旋钮)/ Redis exp(实验),都不重启。
- **一个 Dockerfile 参数化所有服务**:MODULE/PORT build-arg,pom-layer 缓存加速。
